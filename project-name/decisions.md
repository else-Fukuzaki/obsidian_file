## 構造・方針

### プロジェクト管理方法
- **決定日**: 2026-04-14
- **内容**: フロントマッター + Dataview でクライアント別・カテゴリ別に進捗管理
- **理由**: 複数クライアント・複数カテゴリの案件を効率的に追跡できるため
- **影響**: すべてのノートで統一フォーマットを使用

- **決定日**: 2026-04-14

### フロントマッターの統一化
- **決定日**: 2026-04-14
- **フィールド**: 
  - project, client, industry, category
  - status, priority, version
  - deadline, impact

- **理由**: クライアント、業界、カテゴリ、期限、売上への影響を一元管理し、Dataview で多面的に分析できるため
